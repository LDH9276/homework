# 두 번째 회고

## 웹 접근성과 시맨틱 태그의 이해.

이번주는 시맨틱 html 태그를 배우면서 웹 접근성(WAI-ARIA)에 집중적으로 수업했다.
내가 웹 퍼블리셔로 일할 때는 `<header>, <main>, <footer>` 외 맞는 태그들
`<div>와 <p>, <b>, <span>` 위주로 사용했기 때문이었다.

하지만, 시맨틱 코드와 WAI-ARIA에 대입해서 작성하다 보니 정말 새로운 것을 배운 느낌이었다.

```
<div>
  <div>후기</div>
  <div>
    <p>무엇이 중요한지 알게된 <span>시간</span>!</p>
  </div>
  <div>제가 들었던 해외 튜토리얼에 비해 <b>시맨틱한 마크업</b>과 <b>접근성</b> 비중이 크게 다뤄져 좋았습니다. 많은 튜토리얼들이 디자인을 어떻게 구현하는지에만 초점이 맞춰져 있잖아요. 무엇이 중요한지 알게되어 많은 도음이 되었습니다.</div>
  <div>
    <p>★★★★★</p>
    <span>평점</span> <b>5점</b> <i>우정민</i>
  </div>
</div>
```

라는 코드로 지금까지 작성해 왔다면,
다음과 같이 짤 수 있다는 것이다.

```
<div>
  <!-- sr-only는 tailwind에서 쓰는 것으로 숨김 요소 -->
  <h2 class="sr-only">후기</h2>
  <p>
    <em>무엇이 중요한지 알게된 <em>시간</em>!</em>
  </p>
  <p>제가 들었던 해외 튜토리얼에 비해 <strong>시맨틱한 마크업</strong>과 <strong>접근성</strong> 비중이 크게 다뤄져 좋았습니다. 많은 튜토리얼들이 디자인을 어떻게 구현하는지에만 초점이 맞춰져 있잖아요. 무엇이 중요한지 알게되어 많은 도음이 되었습니다.</p>
  <p>
    <!-- aria-hidden은 숨김처리, 불필요한 시각적 요소를 가린다. -->
    <span aria-hidden="true">★★★★★</span>
    <span class="sr-only">평점</span> <b>5점</b> <i>우정민</i>
  </p>
</div>
```

## 향후 수업의 집중할 내용

향후 과제나 퍼블리싱에 대해서는 `<div>, <p>, <span>`에 의존해서  
CSS 스타일링으로만 만드는 것이 아니라, 시맨틱 코드의 집합과  
이를 잘 지키면서 웹페이지를 만드는 능력의 배양일 것 같다.  
또한 html 마크업 검사로 버릇처럼 만든 것들도 틀리게 사용한 것도 많았으니 다시 복습한다는 의미에서  
의미있는 한 주였던 것 같다.
