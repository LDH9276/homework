# 네 번째 회고

## 성장 1: 클레스에 의지하지 않고 사용.

가장 큰 성장은 클래스에 의존해 가령 `.code.black` 같이 뒤의 클래스를 더 붙여 의존하는 것보다
인라인 스타일로 변수를 보내 지정하여 유지보수가 간결하게 할 수 있고 리액트를 사용함으로
프론트엔드 환경에서 쾌적할 수 있도록 작성하는데에 의의가 있다.

## 성장 2 : 다양한 플렉스 / 그리드 / float로 레이아웃 간결하게.

기존에는 보이는 대로 만들었다면 이제는 생각을 하면서 레이아웃을 짤 수 있었다.
그러기에 유지보수 상황에서 다 지우고 다시 작성했다면,
이제는 시멘틱 코드를 이용해 작성을 가능한 것 같다.

하지만, 습관이 습관이라고 논리속성을 잘 이용하지 않고
편한대로 div에 flex속성만 주기로 2년간의 경력으로 가끔 가다 편한 방식으로 구현을 해서,
다음 프로젝트나 포트폴리오 제작 시 이 부분을 상당히 조심스럽게 접근해야 겠다.

## 성장 3 : 미디어 쿼리 등 다양한 선택자로 유지보수를 잘 짜는 CSS.

성장 1 처럼 코드가 지저분해져, 다시는 유지보수를 생각하지 않고 작성한 적이 많았지만,
변수를 사용하다 보니 이 부분과 부모와 자식 사이에 넣을 수 있어 괜히
`.a .b` `.a > .b`와 같이 따로 쓰지 않고 한다는 것 자체에 많은 코드 줄수를 아낄 수 있던 것 같다.
특히 해당 방법으로 과제를 만들어 보니 코드의 줄 수가 3분의 2정도로 엄청 간결하게 짤 수 있다는 이점에
상당히 놀라웠다.

## 앞으로의 방향성

CSS를 능동적으로 짜는 법과 MDN 문서가 상당히 도움이 된 것 같았다.
향후 과제는 변수를 자유자제로 쓰는 것과 block, inline 속성값에 더 깊게 공부할 차례가 남은 것 같다.
또한 자바스크립트에서 능동적으로 바꿀때가 가장 기대될 것 같다.
